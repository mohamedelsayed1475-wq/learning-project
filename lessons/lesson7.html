🏗️ البنية الدلالية (Semantic HTML)

HTML5 أضافت وسوماً "ذكية" لها معنى ودلالة واضحة. تعلّم هذه الوسوم يجعل كودك أنظف، وأفضل لمحركات البحث، وأكثر سهولة في القراءة للمطورين الآخرين.

1. ما معنى "Semantic" (دلالي)؟

الفرق بين كود دلالي وغير دلالي:

❌ كود غير دلالي (قديم)

<div id="header">...</div> <div id="nav">...</div> <div id="main">...</div> <div id="footer">...</div>

المتصفح لا يعرف وظيفة كل div ويعتبرها حاويات فارغة بدون أي معنى دلالي.

✅ كود دلالي (حديث - HTML5)

<header>...</header> <nav>...</nav> <main>...</main> <footer>...</footer>

المتصفح ومحركات البحث وقارئات الشاشة تفهم فوراً وظيفة ومحتوى كل عنصر.

2. الوسوم الدلالية الأساسية

<header> — رأس الصفحة أو القسم
يحتوي عادةً على الشعار، اسم الموقع، قائمة التنقل الرئيسية، أو مقدمة وعنوان المقال/القسم.
<nav> — قائمة التنقل
يُستخدم خصيصاً لاحتواء الروابط الانتقالية الرئيسية والفرعية داخل الموقع. لا تضع كل رابط بالصفحة هنا، بل فقط روابط التنقل الرئيسية.
<main> — المحتوى الرئيسي
يحتوي على المحتوى الفعلي والفريد للصفحة الحالية والذي لا يتكرر في الصفحات الأخرى. يجب كتابته مرة واحدة فقط في الصفحة.
<section> — قسم
يُقسّم المحتوى الطويل إلى أقسام أو فصول منطقية ذات موضوع مشترك. يفضل دائماً أن يبدأ كل قسم بعنوان (h2, h3...).
<article> — مقال أو محتوى مستقل
يمثل كتلة محتوى مستقلة تماماً بذاتها ومفهومة حتى لو تم فصلها ونشرها في مكان آخر (مثل مقال إخباري، منشور مدونة، تعليق مستخدم، أو كارت منتج).
<aside> — محتوى جانبي
يحتوي على معلومات جانبية أو ثانوية ليست أساسية، ولكنها ترتبط بالمحتوى الرئيسي (مثل شريط الروابط الجانبية، الإعلانات، الاقتباسات، أو معلومات الكاتب).
<footer> — ذيل الصفحة أو القسم
يمثل تذييل الصفحة للموقع أو المقال، ويحتوي عادةً على حقوق الملكية الفكرية، روابط الاتصال والتواصل، روابط سياسة الخصوصية، إلخ.

3. هيكل صفحة كاملة باستخدام الوسوم الدلالية

<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
  <meta charset="UTF-8">
  <title>موقعي الإخباري</title>
</head>
<body>

  <!-- رأس الموقع -->
  <header>
    <nav>
      <a href="/">الرئيسية</a>
      <a href="/news">الأخبار</a>
      <a href="/contact">التواصل</a>
    </nav>
  </header>

  <!-- المحتوى الرئيسي للموقع -->
  <main>

    <!-- قسم المقالات الإخبارية -->
    <section>
      <h2>آخر الأخبار</h2>

      <!-- مقال إخباري أول -->
      <article>
        <h3>عنوان الخبر الأول</h3>
        <p>محتوى الخبر هنا...</p>
      </article>

      <!-- مقال إخباري ثاني -->
      <article>
        <h3>عنوان الخبر الثاني</h3>
        <p>محتوى الخبر هنا...</p>
      </article>
    </section>

    <!-- الشريط الجانبي للموقع -->
    <aside>
      <h3>أكثر الأخبار قراءة</h3>
      <ul>
        <li><a href="#">خبر هام 1</a></li>
        <li><a href="#">خبر هام 2</a></li>
      </ul>
    </aside>

  </main>

  <!-- ذيل الموقع -->
  <footer>
    <p>&copy; 2024 موقعي. جميع الحقوق محفوظة.</p>
  </footer>

</body>
</html>
💡 لماذا الـ Semantic HTML غاية في الأهمية؟
• تحسين محركات البحث (SEO): جوجل يفهم هيكل موقعك ويصنفه بشكل أفضل وأسرع.
• سهولة الوصول (Accessibility): تساعد قارئات الشاشة المكفوفين على فهم محتوى موقعك والتنقل داخله بذكاء.
• نظافة الكود: تسهيل قراءة وفهم وتعديل الكود البرمجي للمطورين الآخرين معك بالمشروع.

4. وسوم دلالية إضافية ومهمة

<figure> و <figcaption>

يُستخدمان لتغليف صورة أو رسم توضيحي مع كتابة عنوان أو تعليق نصي يصفها ويكون ملحقاً بها مباشرة.

<figure> <img src="chart.png" alt="رسم بياني"> <figcaption>رسم بياني للمبيعات لعام 2024</figcaption> </figure>

<time>

يُستخدم لتمثيل التواريخ والأوقات بشكل دلالي تفهمه محركات البحث والمتصفحات الذكية.

<time datetime="2024-06-07">7 يونيو 2024</time>

<address>

يُستخدم لعرض معلومات الاتصال بالكاتب أو الشركة المالكة للموقع (مثل البريد، العنوان، رقم الهاتف).

<address>القاهرة، مصر</address>
