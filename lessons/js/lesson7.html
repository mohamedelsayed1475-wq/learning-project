⚡ معالجة الأحداث والتفاعل (Event Handling)

الآن نجعل الموقع حياً ومتفاعلاً! تعلم كيف تستجيب لضغطة زر من المستخدم، أو الكتابة في حقل إدخال، أو إرسال نموذج، لتقديم تجربة حركية تفاعلية.

⚡ ما هي الأحداث (Events)؟

الحدث هو إشارة أو فعل يحدث في النظام (مثل المتصفح)، ويشير إلى أن شيئاً ما قد تم القيام به. أمثلة: ضغط زر الفأرة، تحميل الصفحة، تحريك الفأرة، الضغط على لوحة المفاتيح، أو إرسال نموذج.

💡 قديماً، كان المطورون يكتبون الأحداث في كود HTML هكذا: <button onclick="alert('hi')">. هذه الممارسة سيئة وتجعل الكود غير منظم. الطريقة الحديثة والاحترافية هي ربط الحدث من خلال جافاسكريبت بالكامل.

📞 مسجل ومستمع الأحداث (addEventListener)

الطريقة القياسية والأنظف للاستماع لحدث معين وتطبيق كود عند حدوثه:

// 1️⃣ تحديد الزر من الصفحة
const myButton = document.querySelector('.action-btn');

// 2️⃣ تسجيل مستمع الأحداث
myButton.addEventListener('click', function() {
  // الكود بداخل هذه الدالة (Callback) سيعمل فقط عند نقر الزر
  console.log('تم النقر على الزر بنجاح! 🚀');
});

// 3️⃣ أو كتابتها باستخدام الدالة السهمية (Arrow Function)
myButton.addEventListener('click', () => {
  document.body.classList.toggle('dark-theme');
});

🌟 أشهر أنواع الأحداث في صفحات الويب

const inputField = document.querySelector('#username');
const myForm = document.querySelector('#login-form');

// 1️⃣ حدث 'input': يعمل لحظياً أثناء كتابة المستخدم لأي حرف
inputField.addEventListener('input', (event) => {
  console.log('النص الحالي:', inputField.value);
});

// 2️⃣ حدث 'change': يعمل عندما ينهي المستخدم الكتابة ويخرج من الحقل
inputField.addEventListener('change', () => {
  console.log('تم تأكيد النص');
});

// 3️⃣ حدث 'keydown' و 'keyup': عند الضغط على لوحة المفاتيح
window.addEventListener('keydown', (e) => {
  if (e.key === 'Escape') {
    console.log('تم الضغط على زر Esc لإغلاق النافذة');
  }
});

// 4️⃣ حدث 'submit': عند إرسال نموذج HTML (Form)
myForm.addEventListener('submit', (e) => {
  // سنشرح هذا السطر الهام في القسم التالي
  e.preventDefault();
  console.log('تم إرسال البيانات بدون إعادة تحميل الصفحة!');
});

📂 كائن الحدث (Event Object) وإيقاف السلوك الافتراضي

عند حدوث أي حدث، يمرر المتصفح تلقائياً كائناً يحتوي على معلومات تفصيلية عن هذا الحدث (نستقبله غالباً بـ e أو event). 

const card = document.querySelector('.card');

card.addEventListener('click', (e) => {
  // 1. e.target: يشير للعنصر الحقيقي الذي تم نقره
  console.log('العنصر الذي نقرته هو:', e.target);

  // 2. معلومات أخرى عن الحدث
  console.log('موضع النقر الأفقي:', e.clientX);
  console.log('موضع النقر العمودي:', e.clientY);
});

// 3. e.preventDefault(): منع السلوك التلقائي للمتصفح
const link = document.querySelector('#my-link');
link.addEventListener('click', (e) => {
  e.preventDefault(); 
  // يمنع الرابط من الانتقال لعنوان الويب المحدد له
  // هذا يفيدنا عند الرغبة في تنفيذ كود جافاسكريبت بدلاً من الانتقال
  console.log('لن ينتقل الرابط لأي مكان الآن!');
});

🎓 ما تعلمناه في هذا الدرس

  • ✅ الأحداث هي تفاعلات المستخدم والأفعال التي تقع بداخل الصفحة.
  • ✅ نستخدم addEventListener لربط الأحداث بالعناصر بشكل نظيف.
  • ✅ الأحداث الأكثر شيوعاً هي click و input و submit.
  • ✅ كائن الحدث (Event Object) يُمرر تلقائياً ويحمل معلومات تفصيلية هامة.
  • ✅ e.preventDefault() تمنع السلوك الافتراضي للمتصفح (كإرسال النماذج أو الانتقال بالرابط).
