📱 التصميم المتجاوب واستعلامات الوسائط (Media Queries)

تعلم كيف تجعل موقعك يبدو جميلاً واحترافياً على كافة الشاشات، من الهواتف المحمولة الصغيرة وحتى شاشات الحواسيب الضخمة!

🤔 ما هو التصميم المتجاوب (Responsive Web Design)؟

التصميم المتجاوب هو أسلوب تصميم يهدف لتجعل صفحات الويب تبدو مناسبة وتعمل بشكل مثالي على مختلف الأجهزة والقرارات وشاشات العرض.

💡 قديماً، كان المطورون يبنون موقعاً خاصاً للهواتف (مثل m.facebook.com) وموقعاً آخر للحاسوب. اليوم، نستخدم كود CSS واحداً يتكيف مع حجم الشاشة تلقائياً!

الركائز الأساسية الثلاث للتصميم المتجاوب:

  • 📏 تخطيطات مرنة (Fluid Grids): استخدام النسب المئوية % أو وحدات مثل fr و vw بدلاً من العرض الثابت بالبكسل.
  • 🖼️ صور مرنة (Responsive Images): جعل الصور لا تزيد عن حجم حاويتها بـ max-width: 100%;.
  • 📱 استعلامات الوسائط (Media Queries): تطبيق تنسيقات مختلفة بناءً على حجم الشاشة.

📱 استعلامات الوسائط (Media Queries)

تتيح لك استعلامات الوسائط كتابة شروط معينة في CSS، بحيث لا تطبق هذه التنسيقات إلا إذا تحققت تلك الشروط (مثل حجم شاشة معين). 

/* التنسيق الافتراضي للشاشات الكبيرة (Desktop-First) */
.container {
  width: 80%;
  margin: 0 auto;
  display: flex;
}

.sidebar {
  width: 30%;
  background-color: #f1f5f9;
}

.main-content {
  width: 70%;
}

/* 📱 استعلام الوسائط للهواتف (عرض الشاشة أقل من أو يساوي 768px) */
@media screen and (max-width: 768px) {
  .container {
    flex-direction: column; /* تحويل العناصر لترتيب عمودي بدلاً من أفقي */
    width: 95%;            /* جعل الحاوية أوسع على الشاشات الصغيرة */
  }

  .sidebar, .main-content {
    width: 100%;           /* أخذ العرض الكامل للشاشة */
  }
}

🎯 نقاط التوقف الشائعة (Breakpoints)

نقطة التوقف هي الحجم الذي يتغير عنده تنسيق الصفحة. تختلف نقاط التوقف بناءً على الأجهزة المستهدفة، ولكن الأكثر شيوعاً هي: 

/* 1. للهواتف المحمولة الصغيرة جداً (أقل من 480 بكسل) */
@media screen and (max-width: 480px) {
  h1 {
    font-size: 24px; /* تصغير العناوين لتناسب شاشات الموبايل الصغيرة */
  }
}

/* 2. للأجهزة اللوحية والهواتف الكبيرة (أقل من 768 بكسل) */
@media screen and (max-width: 768px) {
  .nav-menu {
    display: none; /* إخفاء قائمة التنقل لعرض زر همبرغر بدلاً منها */
  }
}

/* 3. للحواسيب المحمولة والشاشات المتوسطة (بين 769 و 1024 بكسل) */
@media screen and (min-width: 769px) and (max-width: 1024px) {
  .card {
    width: 50%; /* عرض كارتين في كل صف */
  }
}

/* 4. للحواسيب المكتبية والشاشات الكبيرة جداً (أكثر من 1200 بكسل) */
@media screen and (min-width: 1200px) {
  .container {
    max-width: 1140px; /* تحديد حد أقصى لعرض المحتوى كي لا يتمدد بشكل قبيح */
  }
}

⚡ نهج الموبايل أولاً (Mobile-First)

أثناء التصميم، يفضل اتباع نهج Mobile-First وهو كتابة الأكواد الخاصة بالهاتف كأكواد أساسية، ثم استخدام min-width لإضافة تنسيقات الأجهزة الأكبر تدريجياً. هذا يجعل الكود أنظف وأسرع في التحميل على الهواتف.

/* 📱 التنسيق الأساسي للموبايل (Mobile-First) */
.grid-layout {
  display: grid;
  grid-template-columns: 1fr; /* عمود واحد في الموبايل */
  gap: 15px;
}

/* 💻 الأجهزة اللوحية وما فوق (أكبر من 768px) */
@media screen and (min-width: 768px) {
  .grid-layout {
    grid-template-columns: repeat(2, 1fr); /* عمودان متساويان */
  }
}

/* 🖥️ شاشات الحاسوب الكبيرة (أكبر من 1024px) */
@media screen and (min-width: 1024px) {
  .grid-layout {
    grid-template-columns: repeat(3, 1fr); /* 3 أعمدة */
  }
}

🎓 ما تعلمناه في هذا الدرس

  • ✅ التصميم المتجاوب يجعل الموقع يعمل بشكل رائع على جميع الشاشات.
  • ✅ @media تستخدم لفحص الحجم وتطبيق خصائص مخصصة له.
  • ✅ max-width يستهدف الشاشات الأصغر من حجم معين (منهج Desktop-First).
  • ✅ min-width يستهدف الشاشات الأكبر من حجم معين (منهج Mobile-First).
  • ✅ استخدام الوحدات المرنة مثل flex و grid يساعد كثيراً في تحقيق التجاوب بسهولة.
